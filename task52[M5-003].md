# Task 52 [M5-003]: Response Caching with Redis

## Description

GET /api/chapters and the leaderboard are called frequently and are expensive to compute. This task implements Redis caching to reduce database and API load.

## Tasks

- [ ] Install cache-manager + Redis adapter in NestJS
- [ ] 5-minute cache for GET /api/chapters (public list)
- [ ] 2-minute cache for global leaderboard
- [ ] Invalidate cache when a chapter is published or updated
- [ ] Cache on-chain certificate queries (Soroban RPC is slow)

## Type

| Priority | Estimate |
|----------|----------|
| Medium | 2 days |

---

## Implementation Plan

### Phase 1: Setup

1. **Install Dependencies**
   ```bash
   npm install cache-manager @nestjs/cache-manager redis
   npm install -D @types/cache-manager
   ```

2. **Add Environment Variables**
   ```env
   REDIS_URL=redis://localhost:6379
   CACHE_TTL_CHAPTERS=300000
   CACHE_TTL_LEADERBOARD=120000
   ```

3. **Create Cache Module**
   - File: `src/cache/cache.module.ts`
   - Configure Redis store with connection pooling

4. **Import in App Module**
   - File: `src/app.module.ts`
   - Import `CacheModule` in imports array

### Phase 2: Chapters Cache

| Endpoint | TTL | Method |
|----------|-----|--------|
| GET /api/chapters | 5 min (300s) | CacheInterceptor |

**Files to Modify:**
- `src/chapters/chapters.controller.ts`
- `src/chapters/chapters.service.ts`

**Invalidation Triggers:**
| Action | Cache Key | Method |
|--------|-----------|--------|
| Publish chapter | chapters:* | chaptersService.create() |
| Update chapter | chapters:* | chaptersService.update() |
| Delete chapter | chapters:* | chaptersService.remove() |

### Phase 3: Leaderboard Cache

| Endpoint | TTL | Method |
|----------|-----|--------|
| GET /podium/leaderboard | 2 min (120s) | CacheInterceptor |

**Files to Modify:**
- `src/podium/podium.controller.ts`
- `src/podium/podium.service.ts`

**Invalidation Triggers:**
| Action | Cache Key | Method |
|--------|-----------|--------|
| Weekly reset | podium:* | podiumService.weeklyReset() |
| Progress update | podium:* | progressService.update() |

### Phase 4: Certificate Cache

| Method | TTL | Caching Strategy |
|--------|-----|-----------------|
| ActaService.verifyCredential() | 10 min (600s) | Manual cache by VC ID |

**Files to Modify:**
- `src/acta/acta.service.ts`

**Reason:** Soroban RPC calls are slow; cache verification lookups
